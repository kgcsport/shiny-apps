# Manifest

Open `README.md` first for the purpose, claim, and recommended workflow. Then use `01_prompt_template.md` to start a prototype and `04_testing_checklist.md` before any live classroom use.

## Root Files

- `README.md`: Overview of the kit, its intended users, its limits, and the recommended workflow.
- `MANIFEST.md`: One-page guide to every file in the kit and what it is for.
- `01_prompt_template.md`: Copyable prompt template for Claude Code, Codex, Cursor, or another agentic AI coding tool, plus a completed public goods game example.
- `02_agents_template.md`: Generic `AGENTS.md` or `CLAUDE.md` instructions that an instructor can copy into a new classroom tool project.
- `03_privacy_workflow.md`: Practical workflow for using AI without sharing student PII.
- `04_testing_checklist.md`: Pre-class testing checklist covering installation, classroom flow, data, failures, and readiness.
- `05_review_form.md`: Short review form for instructors who test the workflow or a generated tool.
- `06_adaptation_guide.md`: Guide for changing rules, displays, stored data, exports, and fake-to-real local data setup.

## Fake Data

- `fake_data/fake_roster.csv`: Thirty fake participants with placeholder IDs and section labels.
- `fake_data/fake_groups.csv`: Ten fake groups of three participants for one demo session.
- `fake_data/fake_choices.csv`: Sample event-level choices for several classroom activity types.
- `fake_data/fake_sessions.csv`: Demo session metadata.
